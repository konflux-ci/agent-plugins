No, you don't necessarily need to add it to both sections, but it depends on your use case:

**`<repositories>`** section:
- Used for project dependencies (libraries your code uses)
- Required if the trusted repository contains your project's dependencies

**`<pluginRepositories>`** section:
- Used for Maven plugins (build tools)
- Required if the trusted repository contains Maven plugins you need

**In practice:**
- If your repository only hosts library dependencies, add it to `<repositories>`
- If it only hosts plugins, add it to `<pluginRepositories>`
- If it hosts both, add it to both sections

Most commonly, you'll only need the `<repositories>` section. You'd only need `<pluginRepositories>` if you're using custom build plugins from that repository.
