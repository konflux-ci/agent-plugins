Not necessarily. It depends on what you're using from that repository:

- **`<repositories>`** — Use this if the repository contains regular project dependencies (JARs, libraries, etc.)
- **`<pluginRepositories>`** — Use this if the repository contains Maven plugins

**In practice:**

1. **If the repository only has dependencies**: Add it only to `<repositories>`
2. **If the repository only has plugins**: Add it only to `<pluginRepositories>`
3. **If the repository has both**: Add it to both sections

For example:

```xml
<repositories>
  <repository>
    <id>my-repo</id>
    <url>https://my-repo.example.com/maven</url>
  </repository>
</repositories>

<pluginRepositories>
  <pluginRepository>
    <id>my-repo</id>
    <url>https://my-repo.example.com/maven</url>
  </pluginRepository>
</pluginRepositories>
```

**Common case:** Most public Maven repositories (like Maven Central) serve both dependencies and plugins, so they're typically configured in both sections. However, if you're adding a custom/private repository and you know it only contains one type, you can skip the unnecessary section.
