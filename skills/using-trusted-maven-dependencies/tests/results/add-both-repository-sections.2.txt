Not necessarily. It depends on what you're using from that repository:

- **`<repositories>`** — for regular project dependencies (artifacts your code directly depends on)
- **`<pluginRepositories>`** — for Maven plugins (tools that Maven uses during the build, like compilers, test runners, etc.)

**Add to both only if:**
- The repository hosts both regular dependencies AND plugins you need

**Add to just one if:**
- The repository only hosts regular dependencies → just `<repositories>`
- The repository only hosts plugins → just `<pluginRepositories>`

**In practice:**
Most third-party repositories (Maven Central, JCenter, etc.) host regular dependencies, so you typically only need `<repositories>`. Plugin repositories are less common—you'd use them if you're pulling a custom Maven plugin from a non-standard location.

If you're unsure, start with just `<repositories>`. Maven will tell you if it can't find a plugin and you'll need to add it to `<pluginRepositories>` as well.
