Not necessarily. Whether you need both sections depends on what you're adding the repository for:

**Use `<repositories>` alone if:**
- You're adding a repository for project dependencies (libraries your code depends on)

**Use `<pluginRepositories>` alone if:**
- You're adding a repository for Maven plugins (build tools, code generators, etc.)

**Use both if:**
- The repository hosts both regular dependencies AND Maven plugins that your build needs

In most cases, you'll only need one or the other. For example:
- A repository hosting a custom Java library → only `<repositories>`
- A repository hosting a custom Maven plugin → only `<pluginRepositories>`
- A repository like Sonatype's snapshot repo that hosts both dependencies and plugins → both sections

If you're unsure, you can start with just `<repositories>` and add `<pluginRepositories>` later if you encounter issues resolving plugins from that repository.
