It depends on what you're sourcing from the trusted repository:

- **`<repositories>`** — where Maven looks for project dependencies (libraries your code depends on)
- **`<pluginRepositories>`** — where Maven looks for Maven plugins (build tools and extensions)

Add the repository to:
- **Just `<repositories>`** if you're only sourcing project dependencies
- **Just `<pluginRepositories>`** if you're only sourcing Maven plugins
- **Both sections** if you're sourcing both types of artifacts from the same repository

In practice, if you're setting up a trusted internal repository, it often makes sense to add it to both sections since it typically serves as a complete replacement for Maven Central and hosts both types of artifacts.

Example structure:
```xml
<repositories>
  <repository>
    <id>trusted-repo</id>
    <url>https://your-repo-url</url>
  </repository>
</repositories>

<pluginRepositories>
  <pluginRepository>
    <id>trusted-repo</id>
    <url>https://your-repo-url</url>
  </pluginRepository>
</pluginRepositories>
```
