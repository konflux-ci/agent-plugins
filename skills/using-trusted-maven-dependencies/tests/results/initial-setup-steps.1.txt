I'll help you configure your Maven project to use a trusted repository instead of Maven Central. Here's how to do it:

## Option 1: Modify `pom.xml` (Project-Level)

Add a `<repositories>` section to your `pom.xml`:

```xml
<project>
  ...
  <repositories>
    <repository>
      <id>my-trusted-repo</id>
      <name>My Trusted Repository</name>
      <url>https://repo.example.com/nexus/content/repositories/releases/</url>
      <releases>
        <enabled>true</enabled>
      </releases>
      <snapshots>
        <enabled>false</enabled>
      </snapshots>
    </repository>
  </repositories>
  ...
</project>
```

## Option 2: Modify `settings.xml` (User/Global Level)

Edit `~/.m2/settings.xml` (or `%USERPROFILE%\.m2\settings.xml` on Windows):

```xml
<settings>
  ...
  <profiles>
    <profile>
      <id>trusted-repo</id>
      <repositories>
        <repository>
          <id>my-trusted-repo</id>
          <name>My Trusted Repository</name>
          <url>https://repo.example.com/nexus/content/repositories/releases/</url>
        </repository>
      </repositories>
    </profile>
  </profiles>
  <activeProfiles>
    <activeProfile>trusted-repo</activeProfile>
  </activeProfiles>
  ...
</settings>
```

## Option 3: Disable Maven Central Completely

To prevent Maven from using Maven Central at all, add this to `pom.xml`:

```xml
<repositories>
  <repository>
    <id>central</id>
    <url>https://repo.maven.apache.org/maven2</url>
    <releases>
      <enabled>false</enabled>
    </releases>
    <snapshots>
      <enabled>false</enabled>
    </snapshots>
  </repository>
  <repository>
    <id>my-trusted-repo</id>
    <url>https://repo.example.com/nexus/content/repositories/releases/</url>
  </repository>
</repositories>
```

## Authentication (if needed)

If your trusted repository requires authentication, add to `settings.xml`:

```xml
<servers>
  <server>
    <id>my-trusted-repo</id>
    <username>your-username</username>
    <password>your-password</password>
  </server>
</servers>
```

**Key differences:**
- **pom.xml**: Project-specific, committed to version control, affects all developers
- **settings.xml**: User/machine-specific, not committed, useful for credentials
- Use `<enabled>false</enabled>` to disable a repository; use `<enabled>true</enabled>` to enable it
