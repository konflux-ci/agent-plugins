# architecture

## Purpose

Directs agents to the two canonical sources of Konflux architecture information and teaches progressive discovery strategies for navigating them efficiently.

## Addresses Common Problems

- Agents proposing features that contradict existing ADRs
- Agents not knowing where to find service boundary definitions
- Reading too much or too little before evaluating feature proposals
- Using only one repo when both are needed for complete understanding

## Key Features

- Maps both repos' directory structures for fast navigation
- Progressive discovery strategy (orient, find service, check ADRs, get details)
- Specific guidance for feature definition refinement and critique
- Common mistakes table
