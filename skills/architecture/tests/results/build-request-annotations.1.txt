## To reset the build pipeline to the latest version

Add this annotation to `metadata.annotations` on the Component:

```yaml
metadata:
  annotations:
    build.appstudio.openshift.io/request: "configure-pac"
```

This triggers Konflux to submit a PR that resets the Tekton pipeline in your git repository to the latest template (without any prior customizations).

---

## To switch to a different pipeline template

Set **both** annotations:

```yaml
metadata:
  annotations:
    build.appstudio.openshift.io/pipeline: '{"name":"<name-of-the-pipeline-to-use>","bundle":"latest"}'
    build.appstudio.openshift.io/request: "configure-pac"
```

| Annotation key | Value |
|---|---|
| `build.appstudio.openshift.io/request` | `"configure-pac"` |
| `build.appstudio.openshift.io/pipeline` | JSON string with `name` and `bundle` fields |

The key annotation that drives the reconfiguration in both cases is:

```
build.appstudio.openshift.io/request: "configure-pac"
```
