The Integration Service is made up of **5 controllers**, documented in the `Sub-components/Controllers` section (lines 176–184):

1. **Build Pipeline Controller** - Monitors completed Component build PipelineRuns, creates Snapshot CRs representing the collection of components to test together, and updates the Component's `Status.LastPromotedImage` to promote images to the Global Candidate List (GCL).

2. **Snapshot Controller** - Watches for new Snapshot CRs, creates PipelineRuns based on IntegrationTestScenarios to test them, updates Snapshot status based on test results, and creates Release CRs when a Snapshot passes all required tests and auto-release is enabled.

3. **Component Controller** - Handles Component CR deletions, regenerating a fresh Snapshot that excludes the deleted component.

4. **Scenario Controller** - Manages IntegrationTestScenario CRs, validates their configurations, and ensures they are properly set up before use.

5. **StatusReport Controller** - Manages Snapshot CRs and their testing status, creating and updating them based on PipelineRun test results to provide status reporting for integration test execution.
