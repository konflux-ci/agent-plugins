Based on the architecture guidance, here's what you should review to give informed feedback on a Konflux release strategy feature proposal:

## 1. **Understand the Release Service Architecture**
Start with the canonical source:
- Read `architecture/core/release-service.md` in the [konflux-ci/architecture](https://github.com/konflux-ci/architecture) repo to understand:
  - Current release service responsibilities and boundaries
  - What CRDs and APIs it owns
  - How it interacts with build, integration, and other services
  - Current release strategy implementation

## 2. **Search for Prior Art in ADRs**
Check the `ADR/` directory for existing decisions on release strategies:
- Look for ADRs discussing release workflows, strategies, or deployment patterns
- A new strategy proposal may contradict, duplicate, or build on existing architectural decisions
- Understanding the "why" behind current design prevents proposing incompatible approaches

## 3. **Review User-Facing Contract**
Check the [konflux-ci/docs](https://github.com/konflux-ci/docs) repo:
- `modules/releasing/` - How release strategies are currently documented to users
- This reveals what user-facing behavior and configuration would need to change
- Helps evaluate the impact on existing workflows and documentation

## 4. **Check Service Interactions**
Look at diagrams and service specs to understand:
- Which other services (build, integration, pipeline, enterprise-contract) the new strategy affects
- Whether the proposal respects existing service boundaries
- If new inter-service communication is needed

## 5. **Key Questions for Your Critique**
- **Scope**: Does the new strategy fit within release-service's responsibility, or does it blur boundaries?
- **Compatibility**: Does it complement or conflict with existing release strategies?
- **Integration**: How does it interact with build, integration, and pipeline services?
- **User impact**: What documentation and configuration changes are required?
- **Design rationale**: Is there an ADR explaining why this strategy should be added?

Would you like help locating specific resources or understanding how a particular aspect of the proposal fits into Konflux's architecture?
