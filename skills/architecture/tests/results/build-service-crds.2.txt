The Konflux build service owns the following CRDs, as listed in the `key_crds` field of `build-service.md`:

1. **`Component`** — The primary input to the Build Service. Controllers monitor Component CRs to create and configure build pipelines (PipelineRun definitions via Pipelines as Code).

2. **`Repository`** — A PaC (Pipelines as Code) CR created or reused during PaC provisioning. The Component CR is set as the owner of the Repository CR.
