The Build Service owns two CRDs, as listed in the `key_crds` field of the frontmatter:

1. **`Component`** — The primary input for the Build Service. Controllers monitor Component CRs to create and configure build pipelines. The Component Build Controller creates PipelineRun definitions based on Component CR changes, and manages per-component service accounts (`build-pipeline-$COMPONENT_NAME`).

2. **`Repository`** — Pipelines as Code (PaC) Repository CRs are created (or reused) during PaC provisioning. The Component CR is set as the owner of the Repository CR.
