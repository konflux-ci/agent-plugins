The key annotation is:

```
build.appstudio.openshift.io/request: "configure-pac"
```

Set this in `metadata.annotations` on your `Component` resource to trigger Konflux to reconfigure the build pipeline. The docs cover two scenarios:

**1. Reset to the latest default pipeline** (line 51):
```yaml
metadata:
  annotations:
    build.appstudio.openshift.io/request: "configure-pac"
```

**2. Switch to a different pipeline template** (lines 97–98):
```yaml
metadata:
  annotations:
    build.appstudio.openshift.io/pipeline: '{"name":"<name-of-the-pipeline-to-use>","bundle":"latest"}'
    build.appstudio.openshift.io/request: "configure-pac"
```

In both cases, setting `build.appstudio.openshift.io/request: "configure-pac"` is what triggers the action. Konflux will open a pull request against your git repository with the updated Tekton pipeline definition (note: any prior customizations will be overwritten, so save them first).
