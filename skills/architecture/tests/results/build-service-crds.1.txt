The Konflux build service owns **two CRDs**, as listed in the `key_crds` field of the document's front matter:

1. **`Component`** — The primary input for the Build Service. Controllers monitor Component CRs to create and configure build pipelines. The Component Build Controller watches for Component CR creation/updates and sets up PipelineRun definitions via Pipelines as Code. It also carries build configuration via annotations (e.g., `build.appstudio.openshift.io/request`, `build.appstudio.openshift.io/pipeline`).

2. **`Repository`** — A Pipelines as Code (PaC) CR that the Component Build Controller creates or reuses during PaC provisioning. The Component CR is set as the owner of the Repository CR.
