The Build Service owns **2 CRDs**, as listed in the `key_crds` frontmatter of `build-service.md`:

1. **`Component`** — The primary input for the Build Service. Controllers watch Component CRs to create and configure build pipelines (PaC provisioning, service account creation, etc.).

2. **`Repository`** — A Pipelines as Code (PaC) CR created/reused by the Component Build Controller during PaC provisioning. The Component CR is set as the owner of the Repository CR.
