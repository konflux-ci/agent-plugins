The answer is clear from the docs. When you want to customize pipeline timeouts in a Konflux integration test, you set `Spec.ResolverRef.ResourceKind` to **`pipelinerun`** (lowercase `r`).

From `/tmp/konflux-docs/modules/testing/pages/integration/creating.adoc:188`:

> If this is your case, you need to set **`Spec.ResolverRef.ResourceKind`** to `pipelinerun` (lower case `r`) within your integration test scenario definition — `pipeline` is the default.

Pipeline timeouts are one of several attributes that can only be defined at the `pipelineRun` level (others being service accounts and workspaces). By setting `ResourceKind: pipelinerun`, the integration service resolves your test as a `PipelineRun` definition instead of a `Pipeline`, giving you access to those run-level fields.
