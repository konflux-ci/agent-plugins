According to the Konflux docs (`modules/testing/pages/integration/creating.adoc:188`), you set **`Spec.ResolverRef.ResourceKind`** to **`pipelinerun`** (lowercase `r`) in your integration test scenario definition.

This is needed when you want to customize things like pipeline timeouts, service accounts, or workspaces — since the default `ResourceKind` is `pipeline`, but `pipelinerun` allows you to define those additional configurations directly in the referenced resource.
