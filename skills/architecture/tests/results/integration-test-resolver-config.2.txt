Set `Spec.ResolverRef.ResourceKind` to **`pipelinerun`** (lowercase `r`).

By default, the integration test uses `pipeline`, but if you need to customize pipeline timeouts, service accounts, or workspaces — attributes that can only be defined on a `PipelineRun` — you must set `resourceKind: pipelinerun` in your `IntegrationTestScenario` spec:

```yaml
spec:
  resolverRef:
    resolver: git
    resourceKind: pipelinerun
    params:
      - name: url
        value: https://github.com/konflux-ci/integration-examples
      # ...
```
