You set `Spec.ResolverRef.ResourceKind` to **`pipelinerun`** (lowercase `r`).

From the docs at `/tmp/konflux-docs/modules/testing/pages/integration/creating.adoc` (line 188):

> If this is your case, you need to set **`Spec.ResolverRef.ResourceKind`** to `pipelinerun` (lower case `r`) within your integration test scenario definition — `pipeline` is the default.

Pipeline timeouts (along with service accounts and workspaces) are attributes that can only be defined on a `PipelineRun`, not a `Pipeline`. So when you need to customize them, you switch `resourceKind` from the default `pipeline` to `pipelinerun`, and your resolved ref must point to a `PipelineRun` definition instead.

Example snippet from the docs:

```yaml
resolverRef:
  resolver: git
  resourceKind: pipelinerun   # <-- this enables timeout customization
  params:
    - name: url
      value: https://github.com/konflux-ci/integration-examples
    - name: revision
      value: main
```
